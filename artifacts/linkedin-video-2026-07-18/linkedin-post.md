# CareerOS LinkedIn Post

Most job-search tools ask one question: how do I sound better?

CareerOS asks a harder one: **can I prove it?**

I am building CareerOS, a proof-first job-search operating system for international students and early-career developers in Germany.

It helps users:

- track applications, statuses, follow-ups, and notes
- keep an eye on student work-hour limits
- analyse job descriptions with deterministic skill extraction
- compare a saved CV with a target job description
- map each required skill to actual evidence, confidence, and a proof task
- create ethical, evidence-backed application angles without inventing experience

The core idea is the **Evidence Map**:

`required skill -> my evidence -> confidence -> proof task`

No evidence means no CV-ready claim.

I wanted to build something more useful than another AI CV generator: a system that helps candidates turn real projects, courses, and work into defensible applications.

Built with Next.js, React, TypeScript, Tailwind CSS, Supabase, PostgreSQL, Zod, React Hook Form, and an optional bring-your-own-key AI layer.

This is a full-stack portfolio project in active development, and I am building it deliberately to learn the stack properly from the ground up.

Demo data shown in the video is fictional.
